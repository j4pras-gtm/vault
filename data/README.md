# Raw Data Sources

This directory contains the raw, unabstracted datasets used to build the LinkedIn AI Brain. These files can be loaded by LLMs or scripts when deep, specific context is required beyond the abstracted patterns in the `/skills/` directory.

## Files

### 1. `profiles_425.json.gz`
- **Description:** Full JSON scrape of 425 top LinkedIn creator profiles.
- **Format:** Gzip-compressed JSON array.
- **Contents:** `publicIdentifier`, `headline`, `summary`, `followerCount`, nested arrays for `positions`, `educations`, `skills`, `certifications`, etc.
- **Usage:** Use when generating highly specific profile rewrites or analyzing career trajectories. Must be decompressed before parsing.

### 2. `posts_index.csv`
- **Description:** Index of 6,759 posts mapped to the 425 creators.
- **Format:** CSV (Comma Separated Values).
- **Contents:** `Date`, `Linkedin Post URL`, `Person identifier` (links to `publicIdentifier` in the JSON), `Person Name`, `PostCount`.
- **Usage:** Use to map a specific post URL back to its author's profile data.

### 3. `posts_scraped.html`
- **Description:** Raw HTML table export of 3,121 scraped posts.
- **Format:** HTML table.
- **Contents:** `postUrl`, `text` (full post body), `likes`, `comments`, `reposts`, `postedAt`.
- **Usage:** Use when full, verbatim post text is needed for few-shot prompting or deep linguistic analysis.

## How the Brain Uses This Data

The abstracted intelligence in `/skills/` and `/docs/rawcontent.md` was generated by joining these three files:
1. `posts_scraped.html` provided the post text and engagement metrics.
2. `posts_index.csv` linked the `postUrl` to the `Person identifier`.
3. `profiles_425.json.gz` provided the author's background and follower count based on that identifier.
